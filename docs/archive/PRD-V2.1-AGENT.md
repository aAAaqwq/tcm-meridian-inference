# 中医经络推理 Agent — 验收 PRD v2.1

> **版本**: v2.1 | **日期**: 2026-03-27 | **作者**: 小a CEO
> **核心**: 1 个 OpenClaw Agent + Skill 推理，输入温度数据，输出健康诊断

## 一、产品定义

### 一句话
输入 24 个温度数据 → 特征工程 → 规则推理 → 输出健康评分 + 经络状态 + 调理建议。

### 架构
```
用户 POST JSON
    ↓
OpenClaw Agent (tcm-meridian)
    ↓ 调用 Skill
推理脚本 (Python)
    ↓ 读取
规则库 (JSON, 源自《六条经络辩证》)
    ↓
输出 JSON (评分 + 状态 + 建议)
```

**只需要 1 个 Agent。** 不搞多 Agent，不搞 SaaS 前端。

## 二、输入规范

### 核心输入：24 个温度值
6 条经络 × 左右各 1 次 = 12 个值（单次测量）  
或 6 条经络 × 左右各 2 次 = 24 个值（两次测量，用于趋势）

### 单次测量输入
```json
{
  "subject": { "id": "张三", "name": "张三" },
  "measurements": {
    "liver":      { "left": 35.2, "right": 36.2 },
    "spleen":     { "left": 35.1, "right": 35.5 },
    "kidney":     { "left": 35.0, "right": 35.9 },
    "stomach":    { "left": 36.0, "right": 35.2 },
    "gallbladder":{ "left": 35.1, "right": 36.0 },
    "bladder":    { "left": 36.1, "right": 35.2 }
  }
}
```

### 两次测量输入（含趋势）
```json
{
  "subject": { "id": "张三", "name": "张三" },
  "measurements": {
    "liver": {
      "left":  [35.2, 35.0],
      "right": [36.2, 36.5]
    },
    "spleen": {
      "left":  [35.1, 35.3],
      "right": [35.5, 35.4]
    }
    // ... 其他经络同理
  }
}
```

- 单个数值 = 单次测量（无趋势）
- 数组 = 两次测量（第2次 - 第1次 = trendDelta）

## 三、特征工程

| 特征 | 计算方式 | 用途 |
|------|---------|------|
| 温差 | `right - left` | 判断左右偏侧 |
| 均温 | `(left + right) / 2` | 判断整体温度水平 |
| 趋势 | `第2次 - 第1次` | 判断变化方向 |
| 阴阳判定 | 左=阴=气=父=过去=脑；右=阳=血=母=未来=心 | 语义映射 |
| 偏侧类型 | left_low / right_low / cross / stable | 状态分类 |

### 阴阳基础映射（源自《六条经络辩证》）
```
左 = 气 = 阴 = 父 = 过去 = 脑 = 身体
右 = 血 = 阳 = 母 = 未来 = 心
```

## 四、规则引擎（源自《六条经络辩证》）

### 4.1 单经判症规则

#### 肝经（中封）
| 状态 | 条件 | 症状/建议 |
|------|------|----------|
| 左低 | left < right 且 left 低于阈值 | 代谢差、气虚、乳房结节/小叶增生、注意三高、血稠、温差大难入睡、血压高、肝阳上亢、气滞血瘀、口干口苦、脂肪肝、解毒差、掉发、屁多、失眠、眼睛干涩 |
| 右低 | right < left 且 right 低于阈值 | 血虚、易掉发、体内有风、预防心脏供血不足、睡眠浅、贫血、低血压、经期短、乏力、胸闷气短 |
| 交叉 | 左右差异显著且交替偏低 | 气血两虚、预防脂肪肝、肝囊肿 |

#### 脾经（商丘）
| 状态 | 条件 | 症状/建议 |
|------|------|----------|
| 左低 | left < right | 血糖偏高、四肢乏力、思虑过重、易疲劳、懒言 |
| 右低 | right < left | 湿气重、脾湿下注大肠（大便粘稠）、经期延长量多 |
| 交叉 | 左右差异显著 | 注意血糖 |

#### 肾经（太溪）
| 状态 | 条件 | 症状/建议 |
|------|------|----------|
| 左低 | left < right | 尿酸高（肾小球过滤差）、尿蛋白、耳鸣、耳背、盗汗、手脚心出汗、怕冷、性欲下降、腰椎颈椎、结石、囊肿、骨质疏松、夜尿尿频、水肿、肾阴虚、体质偏父亲 |
| 右低 | right < left | 肾阳虚、怕冷、四肢需保暖、体质偏母亲 |
| 交叉 | 左右差异显著 | 预防结石、囊肿、女性子宫受过伤 |
| 特殊 | 哪边低哪边听力低 | 听力下降 |

#### 胃经（解溪）
| 状态 | 条件 | 症状/建议 |
|------|------|----------|
| 左低 | left < right | 津液偏少、慢性胃炎 |
| 右低 | right < left | 胃阳温度低、消化不良 |
| 交叉 | 左右差异显著 | 反酸、幽门螺旋杆菌风险、饮食不规律 |
| 温差大 | | 胃胀、反酸、胃痛、十二指肠溃疡 |

#### 胆经（丘墟）
| 状态 | 条件 | 症状/建议 |
|------|------|----------|
| 左低 | left < right | 胆红素偏高（总/间接/直接）、头两侧结节、偏头痛、晨起口干口苦 |
| 右低 | right < left | 胆固醇、甘油三酯偏高、脂肪瘤 |
| 交叉 | 左右差异显著 | 预防结石、囊肿、息肉、胆囊表面粗糙 |

#### 膀胱经（昆仑）
| 状态 | 条件 | 症状/建议 |
|------|------|----------|
| 左低 | left < right | 肩颈腰问题 |
| 右低 | right < left | 肩颈腰和肺 |
| 交叉 | 左右差异显著 | 肠道息肉、女性子宫受过伤 |

### 4.2 组合判症规则

| 组合条件 | 结论 |
|---------|------|
| 肝左低 + 胆左低 | 转氨酶偏高、肝炎 |
| 肾左低 + 膀胱右低，或 肾右低 + 膀胱左低 | 颈椎问题（相反低） |
| 肾左低 + 膀胱左低（或都是右低） | 腰椎问题（同向低） |
| 6条经络中 4 条以上偏右低 | 注意心脏供血 |
| 6条经络中 4 条以上偏左低 | 注意头部供血 |
| 胃最阳，温度应最高 | 胃温应参考值最高，温差大→慢性胃炎 |

### 4.3 五行相生相克
```
相生：木(肝)→火(心)→土(脾)→金(肺)→水(肾)→木(肝)
相克：木(肝)克土(脾)、土(脾)克水(肾)、水(肾)克火(心)、火(心)克肺、金(肺)克肝
```

### 4.4 子午流注
| 时辰 | 经络 | 功能 |
|------|------|------|
| 凌晨1-3点 | 肝 | 睡眠养肝血 |
| 上午7-9点 | 胃 | 促进消化、吃早餐 |
| 上午9-11点 | 脾 | 增加胃肠蠕动、促进代谢废物排泄 |
| ... | | |

### 5.2 健康评分算法
- 基础分 100 分
- 每条经络异常扣分：
  - 左低/右低：-5 分（轻度）、-10 分（中度）、-15 分（重度）
  - 交叉：-8 分
  - 温差大（>1.0°C）：额外 -3 分
- 组合规则命中：每条 -5 分
- 最低分 20 分

### 5.3 风险等级
| 等级 | 分数范围 | 颜色 |
|------|---------|------|
| 优秀 | 85-100 | 🟢 |
| 良好 | 70-84 | 🟡 |
| 注意 | 55-69 | 🟠 |
| 预警 | <55 | 🔴 |

## 六、技术实现

### 6.1 项目结构（精简）
```
~/clawd/projects/tcm-meridian-inference-v2/
├── agent.json                    # OpenClaw Agent 配置
├── SKILL.md                      # 推理 Skill（Agent 调用入口）

│   ├── combination_rules.json    # 组合判症规则

├── scripts/

│   └── test_infer.py             # 验收测试脚本
├── fixtures/
│   ├── case_left_low.json        # 左低型 demo
│   ├── case_right_low.json       # 右低型 demo
│   ├── case_cross.json           # 交叉型 demo
│   ├── case_multi.json           # 多经络失衡 demo
│   └── case_stable.json          # 平稳型 demo
├── docs/
│   └── 六条经络辩证.xlsx          # 原始知识文档
└── README.md
```

### 6.2 Agent 配置
- **agentId**: `tcm-meridian`
- **模型**: `glm-5-turbo`（推理不需要强模型，规则为主）
- **Skill**: 推理 Skill 嵌入 workspace
- **调用方式**: 
  - OpenClaw 内部: `sessions_send(sessionKey="agent:tcm-meridian:main", message=JSON)`
  - 对外 API: Gateway HTTP API

### 6.3 推理流程
1. 接收 JSON 输入
2. 调用 `scripts/infer.py` 执行计算
3. 脚本读取规则库，执行特征工程 + 规则匹配
4. 返回结构化 JSON 输出

### 6.4 与旧项目的关系
- **可复用**: 旧项目的规则库结构设计、测试用例思路
- **可删除**: 旧的 FastAPI 服务代码、兼容层、legacy 支持、n8n workflow
- **新项目独立**: 推翻重做，不受旧代码约束

## 七、验收标准（AC）

### AC-1: Agent 可用
- [ ] Agent `tcm-meridian` 注册在 OpenClaw
- [ ] 可通过 `sessions_send` 接收 JSON 输入
- [ ] 返回符合输出规范的 JSON

### AC-2: 推理正确性（核心）
- [ ] 肝左低 → 输出"代谢差、气虚、血压高"等肝左低症状
- [ ] 肝右低 → 输出"血虚、心脏供血不足"等肝右低症状
- [ ] 肾左低 → 输出"尿酸高、耳鸣、肾阴虚"
- [ ] 胆左低 → 输出"胆红素偏高、偏头痛、口干口苦"
- [ ] 膀胱交叉 + 肾同向低 → 输出"腰椎问题"
- [ ] 4条以上偏右低 → 输出"注意心脏供血"
- [ ] 4条以上偏左低 → 输出"注意头部供血"

### AC-3: 组合判症
- [ ] 肝左低 + 胆左低 → "转氨酶偏高风险"
- [ ] 肾左低 + 膀胱右低（或反向）→ "颈椎问题"
- [ ] 肾左低 + 膀胱左低 → "腰椎问题"

### AC-4: 门店讲解层
- [ ] `storefront.focusHeadline` 直接可念
- [ ] `storefront.talkTrack` 3 句话，无技术标签
- [ ] `clientExplanation` 含"不等同于医疗诊断"
- [ ] 平稳 case 不制造紧张感

### AC-5: 健康评分
- [ ] 所有经络正常 → 评分 85-100（🟢 优秀）
- [ ] 多经络异常 → 评分 <55（🔴 预警）
- [ ] 评分在合理范围内，不会出现负分或超过 100

### AC-6: Demo Case 全过
| Case | 场景 | 预期 |
|------|------|------|
| case_left_low | 肝/肾/膀胱左低 | 左低症状 + 头部供血提示 |
| case_right_low | 脾/胆右低 | 右低症状 + 湿气重 |
| case_cross | 肝交叉 | 气血两虚 |
| case_multi | 多经络失衡 | 组合规则命中 + 预警评分 |
| case_stable | 整体平稳 | 优秀评分 + 克制话术 |

### AC-7: 响应时间
- [ ] 单次推理 < 10s
