# 评分算法说明书

> 版本: v2.1 | 最后更新: 2026-04-29

---

## 概述

本文档详细说明TCM经络推理引擎的评分算法，包括计分规则、扣减逻辑、加分机制等。

---

## 计分流程

```
基础分 (100分)
    │
    ▼
Step 1: 计算单经络扣减
    │   - 轻度异常: -1分
    │   - 中度/高度异常: -2分
    │   - 交叉异常: -2分
    ▼
Step 2: 计算组合规则扣减
    │   - 肾膀胱双交叉: -3分
    │   - 多交叉(>=3): -3分
    │   - 右偏/左偏: -4分
    │   - 供血风险: -4分
    │   - 肿块风险: -4分
    │   - 多失衡(>=4): -4分
    ▼
Step 3: 应用加分规则
    │   - 多经络改善(>=3): +4分
    │   - 部分改善: +2分
    │   - 整体平稳: +3分
    ▼
Step 4: 应用保底分
    │   - 最低不低于50分
    │   - 最高不超过100分
    ▼
最终分数 (50-100分)
```

---

## 扣减规则详解

### 单经络扣减

| 状态 | 扣减 | 说明 |
|------|------|------|
| 轻度异常 | -1分 | 稍微偏离正常值 |
| 中度异常 | -2分 | 明显偏离正常值 |
| 高度异常 | -2分 | 严重偏离正常值 |
| 交叉异常 | -2分 | 左右不平衡 |

### 组合规则扣减

| 规则 | 条件 | 扣减 | 说明 |
|------|------|------|------|
| 肾膀胱双交叉 | 肾交叉 AND 膀胱交叉 | -3分 | 泌尿系统双重问题 |
| 多交叉 | 交叉经络>=3个 | -3分 | 多个经络交叉 |
| 右偏 | 右侧低>=4个 | -4分 | 整体右偏低 |
| 左偏 | 左侧低>=4个 | -4分 | 整体左偏低 |
| 心脑供血风险 | 心脑供血组合命中 | -4分 | 心脑血管风险 |
| 头部供血风险 | 头部供血组合命中 | -4分 | 头部供血不足 |
| 颈腰生殖风险 | 颈/腰/生殖组合命中 | -3分 | 相关部位风险 |
| 肿块风险 | 肝脾肿块组合命中 | -4分 | 潜在肿块风险 |
| 多失衡 | 异常经络>=4个 | -4分 | 整体失衡严重 |

---

## 加分规则详解

| 规则 | 条件 | 加分 | 说明 |
|------|------|------|------|
| 多经络改善 | 改善经络>=3个 | +4分 | 显著好转 |
| 部分改善 | 1-2个经络改善 | +2分 | 有所好转 |
| 整体平稳 | 无交叉 AND 异常<=2 | +3分 | 状态稳定 |

## 分数等级

```
100分: ████████████████████ 完美
 90分: █████████████████▌   优秀 (整体状态较好)
 75分: ██████████████▌      亚健康 (轻度失衡)
 60分: ███████████▌         需注意 (中度失衡)

```

### 等级映射

| 分数区间 | 等级 | 建议 |
|---------|------|------|
| 90-100 | 整体状态较好 | 继续保持当前状态 |
| 75-90 | 轻度失衡 | 注意生活习惯，适当调理 |

| 50-60 | 需重点关注 | 必须就医，积极治疗 |

## 计算示例

### 示例1: 轻微问题
```
场景: 1个经络轻度异常
基础分: 100
单经络轻度异常: -1
其他: 无

计算: 100 - 1 = 99
保底检查: 99 >= 50 ✓
最终分数: 99分
等级: 整体状态较好
```

### 示例2: 中度问题

```
场景: 心脑供血风险 + 3个经络异常
基础分: 100
单经络异常: -1 * 3 = -3
心脑供血风险: -4

计算: 100 - 3 - 4 = 93
保底检查: 93 >= 50 ✓
最终分数: 93分
等级: 整体状态较好
```

### 示例3: 严重问题

```
场景: 6个交叉 + 多失衡 + 肿块风险
基础分: 100
单经络交叉: -2 * 6 = -12
多交叉(>=3): -3
多失衡(>=4): -4
肿块风险: -4
总和扣减: -23

计算: 100 - 23 = 77
保底检查: 77 >= 50 ✓
最终分数: 77分
等级: 轻度失衡 (亚健康)
```

## 配置说明

评分规则存储在 `rules/score_rules.json` 中：

```json
{
  "version": "2.1",
  "base_score": 100,
  "min_score": 50,      // 保底分
  "max_score": 100,     // 最高分
  "deductions": [...],  // 扣减规则
  "bonuses": [...]      // 加分规则
}
```

### 修改评分

如需调整扣分力度，修改对应规则的 `score` 值：

```json
// 让交叉问题更严格
{ "rule_id": "single_meridian_cross", "score": -3.0 }

// 让心脑供血问题更宽松
{ "rule_id": "heart_supply_hit", "score": -3 }
```

修改后重启服务即可生效。
